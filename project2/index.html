Project 2: Basic Build

Compared to the web experiences and app interactions we have today, the early web (web 2.0) looks pretty basic. Internet technologies continually evolve, becoming more sophisticated and complex. But at least one aspect of the web has remained essential from its earliest days until now: the hyperlink, connecting one document to another.

In this project, you will explore the design possibilities of basic web code (HTML), focusing on the interactive possibilities of the hyperlink while creating an experience that is basic and brutalist, down to its very roots.


Topics Covered
  • Basic HTML
  • Brutalist Web
  • Wireframes
  • User Journeys

Instructions
  • Create a website with multiple pages linked together, resulting in a well-considered interaction between the user and the computer. The primary emphasis of your design should be on the interactions created by your links. However, visual design is also important within the limits of what our basic code levels allow. The site should lead users through an experience with a clear start and end, allowing users to feel like they are doing something, that the computer is responding accordingly, and that the user’s choices actually affect the outcome.
  • Begin with the word “basic.” Think of all the synonyms that may arise from word association. A prevalent synonym may be “mid.” An old-school synonym might be “plain” or “run-of-the-mill.” This will serve as brainstorming for your content.
  • The following are possible concepts for your site:
    • Tutorial to accomplish a task where each step of the process has multiple choices (not just one or two options)
    • Game where users choose different options leading to different possible endings
    • Quiz or test
    • Basic version of a realistic app/web service
    • A photo album or storybook
  • Your site should meet the following specifications to create a strong interaction:
    • Minimum 30 pages (html files)
    • After each user action (click), incorporate some feedback to make it clear the computer “understood” the action
    • Pages should avoid single-link pages as much as possible, as this does not give the user a sense of choice, and instead focus on giving multiple options for each possible action
    • Incorporate sensible navigation: make it clear how to select one option vs another, and consider adding links going back one step, back to the start, etc.
    • Do not use CSS, inline styles, or JavaScript. Do not copy code from outside sources in this or other projects for this class. For this project, the only visual styling you are allowed to do should be from images you put in. Again, while visual design does matter, the primary focus is on your content and use of links to create an experience. A project that has very interesting use of links and weak typography will receive a higher grade than a project with better typography but uninspired interactions (and highest grades will go to students who do both).
    • Finally, code this project from scratch in a plain text editor like Sublime Text or Text Wrangler.

Deliverables

  • Working HTML pages
